# Sistema de Gerenciamento de Tarefas

## Visão Geral

Este projeto consiste no desenvolvimento de um sistema simples de gerenciamento de tarefas pessoais.

A aplicação deve permitir criar, visualizar, atualizar e concluir tarefas, simulando um cenário real de aplicação CRUD, com foco em clareza de código, organização e entrega funcional.

---

## Objetivos de Aprendizado

### Hard skills
- Modelagem básica de dados
- Implementação de operações CRUD
- Desenvolvimento de APIs REST
- Estruturação de projetos backend

### Soft skills
- Organização do trabalho
- Clareza na comunicação técnica
- Gestão de escopo e prioridades

---

## Escopo do Projeto

### Incluído no escopo
- Criação de tarefas com título e descrição
- Listagem de todas as tarefas
- Atualização de informações de uma tarefa
- Marcação de tarefas como concluídas
- Exclusão de tarefas

### Fora do escopo
- Autenticação e autorização de usuários
- Interface gráfica avançada
- Compartilhamento de tarefas entre usuários
- Notificações ou lembretes automáticos

---

## Requisitos Funcionais

- O sistema deve permitir a criação de uma tarefa contendo título e descrição.
- O sistema deve permitir a listagem de todas as tarefas cadastradas.
- O sistema deve permitir a atualização dos dados de uma tarefa existente.
- O sistema deve permitir marcar uma tarefa como concluída.
- O sistema deve permitir a exclusão de uma tarefa.

---

## Requisitos Não Funcionais

- O código deve estar organizado de forma clara e legível.
- O projeto deve conter instruções básicas para execução local.
- A aplicação deve ser executável localmente sem dependências externas obrigatórias.

---

## Tecnologias Sugeridas

**Obrigatórias**
- Node.js — ambiente de execução da aplicação

**Opcionais**
- Express — framework para criação da API
- SQLite — banco de dados leve para persistência local
- TypeScript — tipagem estática para maior segurança

---

## Critérios de Conclusão

O projeto será considerado concluído quando:
- Todos os requisitos funcionais estiverem implementados.
- A aplicação puder ser executada localmente sem erros.
- O repositório contiver um README explicando como rodar o projeto.
- O código estiver versionado em um repositório Git.

---

## Avaliação da Entrega

A entrega será avaliada com base em:
- Atendimento completo aos requisitos funcionais.
- Qualidade e organização do código.
- Coerência entre o escopo definido e a implementação.
- Clareza das decisões técnicas documentadas.

---
